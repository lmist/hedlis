* What cloak is

=cloak= is a companion CLI for [[https://github.com/jackwener/opencli][OpenCLI]], powered by [[https://github.com/Kaliiiiiiiiii-Vinyzu/patchright/][Patchright]], and it always loads the pinned [[https://github.com/jackwener/opencli/releases/download/v1.6.8/opencli-extension.zip][OpenCLI extension archive]].

The job is simple:

- keep the automation browser headless unless you explicitly ask for a window
- let you pull cookies out of a throwaway Chrome profile only for the current run
- give OpenCLI a clean browser context that is not your day-to-day visible browser

* Install

1. Clone

#+begin_src bash
git clone https://github.com/lmist/cloak.git
cd cloak
#+end_src

2. Install

#+begin_src bash
npm install -g @jackwener/opencli
npx skills add jackwener/opencli
npm install
npx patchright install chromium
npm run build
#+end_src

=cloak= is a companion to [[https://github.com/jackwener/opencli][OpenCLI]], so installing OpenCLI first is part of the normal setup.

The OpenCLI skills are worth installing for Claude Code or Cursor. They give the agent the right language and workflows to use OpenCLI well, adapt to new requests, and build or refine adapters instead of treating every site or task as a one-off.

* Usage

Browse available Chrome profiles and cookie-bearing sites:

#+begin_src bash
cloak profiles list
#+end_src

In a real terminal this opens a searchable, colorized picker. In non-TTY environments it prints a plain-text report grouped by profile.

Launch a clean browser:

#+begin_src bash
cloak run
#+end_src

Launch with runtime Chrome cookies using the interactive picker:

#+begin_src bash
cloak run --cookies-from-browser chrome
#+end_src

Launch with an explicit site URL and profile:

#+begin_src bash
cloak run --cookies-from-browser chrome --cookie-url https://x.com --chrome-profile "Default"
#+end_src

If you want a visible browser window instead of headless mode:

#+begin_src bash
cloak run --window
cloak run -w
#+end_src

For local development:

#+begin_src bash
npm test
npm run typecheck
make ci
just ci
#+end_src

* Storage

Persistent state is limited to the pinned extension archive cache:

- =~/.cache/cloak/opencli-extension.zip=

Runtime-only state stays ephemeral:

- the extracted extension directory is created in the OS temp directory for each run
- the Patchright user profile is created in the OS temp directory for each run
- cookies are never written to disk by =cloak=

* Extension bootstrap

=cloak= uses a package postinstall warm-up plus a runtime repair check.

Postinstall tries to cache the pinned extension archive at:

#+begin_example
~/.cache/cloak/opencli-extension.zip
#+end_example

If that warm-up fails, =cloak run= repairs the cache automatically before Patchright starts. The browser launches with:

- =--disable-extensions-except=<extracted-extension-dir>=
- =--load-extension=<extracted-extension-dir>=

* Cookie flow

=cloak= only reads Chrome cookies at runtime and injects them directly into the running Patchright context.

When you run =cloak run --cookies-from-browser chrome= in a TTY, =cloak= can open the profile/site picker for you. When you already know the exact target, pass =--cookie-url= directly.

* One sharp edge

Chrome cookie extraction depends on =chrome-cookies-secure=, and that tool can collapse same-name cookies across different paths or subdomains before =cloak= sees them. If a login still fails after injection, that is the first thing to suspect.
