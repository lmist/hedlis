* new tool who dis

=cloak= is a companion CLI for [[https://github.com/jackwener/opencli][OpenCLI]], powered by [[https://github.com/Kaliiiiiiiiii-Vinyzu/patchright/][Patchright]], and it always loads the pinned [[https://github.com/jackwener/opencli/releases/download/v1.6.8/opencli-extension.zip][OpenCLI extension archive]].

> It runs a headless browser that imports OpenCLI's extension and your choice of cookies for funsies with opencli.

The job is simple:

- keep the automation browser headless unless you explicitly ask for a window
- let you copy cookies out of a throaway Chrome profile
- give OpenCLI a clean browser context that is not your day-to-day visible browser

lone

#+begin_src bash
git clone https://github.com/lmist/cloak.git
cd cloak
#+end_src

2. Install

#+begin_src bash
npm install -g @jackwener/opencli
npx skills add jackwener/opencli
npm install
npx patchright install chromium
npm run build
#+end_src

=cloak= is a companion to [[https://github.com/jackwener/opencli][OpenCLI]], so installing OpenCLI first is part of the normal setup.

The OpenCLI skills are worth installing for Claude Code or Cursor. They give the agent the right language and workflows to use OpenCLI well, adapt to new requests, and build or refine adapters instead of treating every site or task as a one-off.

During =npm install=, =cloak= asks for consent before creating =~/.config/cloak=. If you answer yes, it prepares:

- =~/.config/cloak/cookies/=
- =~/.config/cloak/extensions/=

and downloads the pinned OpenCLI extension archive there so the CLI is ready to run.

If install is non-interactive, =cloak= skips that bootstrap step and finishes setup later when you run it.

3. =cloak profiles list=

#+begin_src bash
cloak profiles list
#+end_src

Typical output:

#+begin_example
Default: Personal
Profile 1: Work
Profile 3: Throwaway
#+end_example

4. Example: =cloak cookies import --browser chrome --url https://x.com --chrome-profile "Default"=

#+begin_src bash
cloak cookies import --browser chrome --url https://x.com --chrome-profile "Default"
#+end_src

This writes the imported cookies to =~/.config/cloak/cookies/x.com.json= by default.

5. =cloak run=

#+begin_src bash
cloak run
#+end_src


If you want a visible browser window instead of headless mode:

#+begin_src bash
cloak run --window
cloak run -w
#+end_src

For local development:

#+begin_src bash
npm test
npm run typecheck
make ci
just ci
#+end_src

* opencli extension bootstrap

=cloak= uses a package postinstall step plus a runtime repair check.

At install time, =src/install-extension.ts= ensures the pinned OpenCLI extension archive exists at:

<NOT TRUE ANYMORE>
#+begin_example
~/.config/cloak/extensions/opencli-extension.zip
#+end_example

At runtime, =cloak run= validates that archive again before Patchright starts. If the file is missing or invalid, it re-downloads it and only then launches the browser with:

- =--disable-extensions-except=<extracted-extension-dir>=
- =--load-extension=<extracted-extension-dir>=
